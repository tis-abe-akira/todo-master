# Requirements Document

## Project Description (Input)
AI（LangChain+Gemini）で、入力されたTODOからサブタスクに分解します。その際に、最大のポイントですが、常におうぎょうな一般論や原理原則を振りかざして話をおおごとにしてください。
例えば、牛乳を買いに行くと言った場合に、栄養素がどうのこうの、乳製品アレルギーがどうのこうのと一般原則にまで引き上げて到底実行不可能な形にしてください！
プロジェクトタスクであれば、PMBOKにてらしてプロジェクトチャーターを作るとか、
なんらかのタスクの場合費用対効果分析、競合分析など薄っぺらいコンサルが言い出すような理屈をならべてとにかく話をおおごとにしてください。そのようなプロンプトを期待しています。
サブタスク分解はボタンをUIから明示的に押して、TODOの下部に並べましょう。だいたい、TODO１つにつき5~6個の大仰なサブタスクに分解してください。
